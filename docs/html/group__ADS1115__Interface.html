◆ ads1115_i2c_read()

bool ads1115_i2c_read ( uint8_t i2c_addr,
uint8_t reg_addr,
uint8_t * data,
uint8_t len )

Platform-specific I2C read implementation.

Note
Example for Arduino Wire:
Wire.beginTransmission(i2c_addr);
Wire.write(reg_addr);
Wire.endTransmission(false);
Wire.requestFrom(i2c_addr, len);
for(int i=0; i<len; i++) data[i] = Wire.read();
return true;

This function must be assigned to ads1115_i2c_read_t in the handle. It typically follows the "Restart" or "Combined" I2C message format:

  1. Write the register address to the device.
  2. Repeated start.
  3. Read the requested number of bytes.
Parameters
i2c_addrADS1115 7-bit I2C slave address.
reg_addrTarget register address to read from.
[out]dataPointer to the buffer where received data will be stored.
lenNumber of bytes to read (usually 2).
Returns
true if read was successful, false otherwise.

◆ ads1115_i2c_write()

bool ads1115_i2c_write ( uint8_t i2c_addr,
uint8_t reg_addr,
const uint8_t * data,
uint8_t len )

Platform-specific I2C write implementation.

Note
Example for STM32 HAL:
return (HAL_I2C_Mem_Write(&hi2c1, i2c_addr << 1, reg_addr, I2C_MEMADD_SIZE_8BIT, (uint8_t*)data, len, 100) == HAL_OK);

This function must be assigned to ads1115_i2c_write_t in the handle. It follows the standard I2C write protocol:

  1. Start condition
  2. Send Slave Address (Write bit)
  3. Send Register Address
  4. Send Data bytes (MSB first)
  5. Stop condition
Parameters
i2c_addrADS1115 7-bit I2C slave address.
reg_addrTarget register address inside the ADS1115.
dataPointer to the buffer containing data to write.
lenNumber of bytes to be written (usually 2 for 16-bit registers).
Returns
true if write was successful, false otherwise.
